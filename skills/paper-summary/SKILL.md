# Paper Summary — One-Page Visual Summary

把论文核心内容组织成一页高度浓缩的视觉摘要。

## 核心哲学

### 一页原则
一张图必须回答四个问题：
1. **做什么？** — 核心问题和目标
2. **怎么做？** — 方法一句话
3. **效果如何？** — 1-3 个关键数据
4. **为什么重要？** — 一句话贡献

### 不做的事
- ❌ 把论文翻译成中文贴上去
- ❌ 超过一页的内容
- ❌ 信息密度过低（大标题 + 小图 = 浪费）
- ❌ 要求读者滚动才能看完

## 三种模式

### Mode 1: infographic — 信息图

竖版长图，适合社交媒体、公众号、群分享。

```
┌─────────────────────┐
│  🏛️ 标题 + 一句话    │
├─────────────────────┤
│  📌 核心问题         │
│  💡 方法一句话        │
├─────────────────────┤
│  ⚙️ 方法流程简图     │
│  （水平流程，简短）    │
├─────────────────────┤
│  📊 核心数据（大数字） │
│  2-3 个最亮眼的数据   │
├─────────────────────┤
│  🎯 核心贡献（3点）   │
│  📚 论文信息          │
└─────────────────────┘
```

### Mode 2: card-summary — 卡片摘要

1:1 方卡，适合文献管理、Notion、Zotero。

```
┌──────────────────────┐
│  📄 论文标题（1行）    │
│  ✍️ 作者 · 年份 · 源  │
├──────────────────────┤
│  📌 问题              │
│  (1行)                │
├──────────────────────┤
│  ⚙️ 方法              │
│  (2-3行)              │
├──────────────────────┤
│  📊 关键结果          │
│  KPI: XX / YY / ZZ   │
├──────────────────────┤
│  💡 一句话贡献         │
└──────────────────────┘
```

### Mode 3: timeline-figure — 演进图

16:9 横版，适合 README 首图、PPT 插入。

```
┌──────────────────────────────────────────┐
│           研究演进/方法流程                │
│                                          │
│  Step1 ─→ Step2 ─→ Step3 ─→ Step4       │
│  输入   核心处理   关键创新   输出         │
│  (图)   (图)      (图)      (图)          │
│                                          │
│  关键标注：[本论文的创新点高亮]           │
└──────────────────────────────────────────┘
```

## 工作流

### Step 1: 分析论文 → 提取摘要内容

读论文后提取：
- **一句话主张**（不超过 25 字）
- **核心问题**（不超过 30 字）
- **方法一句话**（不超过 40 字）
- **关键数据**（1-3 个最有说服力的数字）
- **核心贡献**（3 个要点，每个 10 字以内）

### Step 2: 确认

| 确认项 | 选项 |
|--------|------|
| 模式 | infographic / card-summary / timeline-figure |
| 语言 | 中文 / English / 双语 |
| 重点 | 方法驱动？结果驱动？概念驱动？ |
| 是否使用真实素材 | 允许从论文截图/用户提供 |

### Step 3: 写 Prompt

按模式和 prompt 模板生成。每页信息必须**高度浓缩**——不能有无关文字。

信息密度规则：
- infographic：6-8 个信息块，总文字 ≤ 200 字
- card-summary：4-5 个信息块，总文字 ≤ 120 字
- timeline-figure：4-6 个节点，每个节点 ≤ 15 字

### Step 4: 生成

调用生图后端，按指定模式生成一页图片。

### Step 5: 输出

- `summary-brief.md`
- `prompt.md`
- `images/summary-infographic.png`（或 card-summary / timeline-figure）
- `README.md` 插入代码（如需要）
