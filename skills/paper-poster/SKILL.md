# Paper Poster — AIGC Academic Conference Poster

把论文内容组织成一张完整的学术海报图。

## 核心哲学

### 海报不是论文的缩放版

- ✅ 一张海报讲清楚"一个核心故事"
- ✅ 视觉层级驱动阅读顺序（先看什么、再看什么）
- ✅ 文字只是辅助，图和数据才是主角
- ❌ 不是把论文截图拼在一起
- ❌ 不是把论文文字硬塞进模板框

### 一张好海报的标准

> 一个路过的人，3 米外能看出主题，3 秒内能说出核心贡献，3 分钟内能理解方法。

---

## 输出规格

| 风格 | 尺寸 | 宽高比 | 适合 |
|------|------|--------|------|
| conference-wide | 48×36 in (1219×914 mm) | 4:3 | 标准学术会议展板 |
| defense-poster | 36×48 in (914×1219 mm) | 3:4 | 答辩/毕设竖版海报 |
| research-showcase | 48×36 in | 4:3 | 课题展示、开放日 |

分辨率：300dpi（可打印质量），输出 PNG + PDF。

## 工作流

### Step 1: 输入分析

接受：
- arXiv/DOI/URL
- PDF 路径
- 课题描述 + 多篇论文
- 已有方法图 + 文字

提取：
- 标题、作者、单位
- 核心问题（1 句话）
- 方法流程（输入→处理→输出）
- 关键结果（1-3 个最有说服力的数据）
- 核心贡献（3 个要点）

### Step 2: 确认

必须确认：
1. **海报风格**：conference-wide / defense-poster / research-showcase
2. **内容重点**：方法为主？结果为主？应用为主？
3. **是否使用真实素材**：论文图、实验截图、代码截图
4. **尺寸/格式**：默认 300dpi，是否要 600dpi 打印版
5. **语言文字**：中文 / English / 双语

### Step 3: 设计 Brief

输出 `poster-brief.md`：

```markdown
Style: conference-wide
Size: 48×36 in, 300dpi
Language: English

Core message: [一句话核心主张]

Visual hierarchy:
1. Title — bold, top banner
2. Method figure — center, 40% area
3. Key results — right column, data callouts
4. Problem & conclusion — left column

Color palette: academic blue + neutral gray
Typography: sans-serif titles, serif body

Source visuals:
- Figure 2 from PDF (method architecture) — main panel
- Table 1 data — simplified to top-3 numbers
```

### Step 4: 写 Prompt

按 paper-deck 的 prompt 体系，为整张海报写一个大 prompt。

Prompt 结构参考 `references/prompt-template.md`。

### Step 5: 生成

调用生图后端生成整张海报。如果生图后端不支持超高分辨率，采用分块策略：

1. 先生成 4K 参考图
2. 再用超分/分块拼接达到 300dpi

### Step 6: 输出

- `poster-brief.md`
- `prompt.md`
- `images/poster-preview.png`（预览用，4K）
- `output/poster.pdf`（可打印，300dpi）

### Step 7: 返修

| 用户说 | 操作 |
|--------|------|
| "文字太小" | 改 prompt 增大字号限制 |
| "结果区域再加一个图" | 改 brief 布局，重写 prompt |
| "换成竖版" | 改尺寸 + 调整布局 |
| "联系方式加上" | 追加到底部通栏 |
